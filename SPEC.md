# 長照服務品質管理系統（JOY）雛形規格書

版本：v0.1（Prototype）  
建立日期：2026-02-24  
範圍：前端雛形（靜態頁面），可直接掛載 VS Code 編輯

---

## 1. 目標與定位
本雛形用於快速呈現整體介面風格與操作流程，包含：
- 左側功能選單（主/次類別）
- 右側主要操作區（查詢頁 / 管理表單頁）
- 以 Mock Data 模擬查詢與帶入資料，便於後續接 API 或整合 Angular/Spring Boot

---

## 2. 版面與導覽結構

### 2.1 Layout
- 左側：功能選單（Sidebar）
- 右側：主要操作畫面（Main Content）
  - 上方固定 Topbar（系統名稱 + 使用者資訊）
  - 中央內容區（查詢卡片、表格、表單）

### 2.2 選單階層
- 系統固定抬頭：**新竹縣智慧福利服務躍升**（Topbar 顯示，不隨選單變動）
- 主類別：**長照服務品質管理**（可展開）
  - 次類別：**個案管理**（查詢頁：`index.html`）
  - 次類別：**統計報表**（Prototype 預留，先顯示提示訊息）

---

## 3. 頁面清單（不同檔案）

### 3.1 查詢頁（index.html）
**功能：**查詢條件輸入 + 結果清單呈現

#### 3.1.1 查詢條件（共 5 項，版面依需求排列）
1. **單位名稱**（第一條，文字輸入，模糊比對）
2. **評估日期起迄**（第二條，兩欄：起/迄；民國 `YYY/MM/DD`）
3. **個案姓名**（第三條左側，文字輸入，模糊比對）
4. **縣市鄉鎮**（第三條右側，下拉；縣市固定新竹縣 + 鄉鎮清單）
5. **訪視人員**（第四條左側，下拉選單，可選擇人員）

#### 3.1.2 結果清單欄位
表格欄位如下：
- 單位名稱
- 個案姓名
- 評估日期 + 約訪時間（同欄顯示）
- 個案電話
- 個案居住鄉鎮
- **訪視人員**
- 功能（按鈕：`管理`）

#### 3.1.3 互動行為
- 點擊 `管理`：
  - 導向 `form.html?id=<recordId>`
  - 以 URL Query 參數傳遞資料識別碼
  - 表單頁以該 id 讀取 mock 資料並呈現

---

### 3.2 管理表單頁（form.html）
**功能：**呈現與編輯單筆個案資料（Prototype 先做前端呈現與基本互動）

#### 3.2.1 表單欄位（最新定義：一行兩欄為主）
- 個案姓名
- 個案身分證（輸入框 + 右側「帶入」按鈕）
- 住址（同一行）
  - 縣市（下拉；Prototype 固定新竹縣）
  - 鄉鎮（下拉：新竹縣鄉鎮）
  - 村里（下拉：依鄉鎮動態更新）
  - 鄰
  - 詳細住址
- 生日（日期點選）
- 評估（下拉：初評、一般複評、出準初評、短期間重複評估、住宿式機構）
- 訪視人員（下拉，表單最下面）
- 訪視資料（區塊，位於訪視人員下方）
  - 表格欄位：評估日期、約訪時間、單位名稱、訪視人員、功能
  - 權限規則（Prototype）：若該列訪視人員 = 上方選擇的訪視人員 → 顯示「編輯」；否則僅顯示「檢視」

> 已取消欄位：約訪時間、評估日期、單位名稱（可下拉可填寫）、申請外看 1966 名冊。

#### 3.2.2 上方操作按鈕
- 訪視紀錄（Prototype：顯示提示訊息）
- 返回（回到查詢頁）
